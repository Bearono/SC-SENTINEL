Multi-Agent 架构
Agent A 识别第三方依赖 CVE，Agent B 审计源码语义，Agent C 汇总生成结构化报告与 PDF。

Docker 隔离沙箱
每次 Fuzzing 在独立容器内执行，完成后自动销毁。eBPF 运行在内核层，对被测程序完全透明。
How It Works

两阶段双机制，从静态到内核

Multi-Agent 静态审计 → 动态 AFL++ 验证 → eBPF 内核事件捕获，三层递进，证据链完整。

01
Agent A
Dependency Risk
识别 CMakeLists.txt 中第三方库，比对 NVD/OSV，输出 CVE 风险摘要。
02
Agent B
Semantic Audit
按函数粒度切片，识别 UAF / 堆溢出 / Double Free，输出漏洞位置和触发条件。
03
eBPF + AFL++
Dynamic Verify
Docker 沙箱 AFL++ 模糊测试，uprobe 挂载 malloc/free，内核层捕获 UAF 事件。
04
Agent C
Report Synthesis
整合所有输出，生成含修复建议的 JSON 结构化报告与可下载 PDF。
Why SENTINEL

不只是静态扫描，而是可触发的证据

对比纯 LLM 代码审计，SENTINEL 在每个核心维度都提供了更强的保障

维度纯 LLM 代码审计SENTINEL
